================================================================================
ARCHITECTURE
================================================================================

Module Structure:
  common.py              - Fixed-width parsing utilities
                         - FixedWidthParser class with extraction methods
                         - Shared header parsing, date formatting, amount parsing
                         
  ep_transactions.py     - Labeled field format parsing (EP-705, EP-707, EP-727)
                         - Handles two-column print-ready layout
                         - Normalizes raw data to standard field keys
                         
  ep_summaries.py        - Tabular summary report parsing
                         - Batch/file/CPD/run aggregation levels
                         - Currency-based summaries
                         
  ep_settlement.py       - 4600 record parsing (EP-746)
                         - VSS sub-report structure detection (EP-747)
                         - Settlement hierarchy extraction
                         
  ep_misc.py             - Key-value pair parsing (EP-756)
                         - Tabular index parsing (EP-999)
                         - Text message handling (EP-750)
                         
  ad_raw.py              - Fixed-width record type detection
                         - Transaction grouping by 0500 boundaries
                         - Per-record-type field extraction
                         
  package_parser.py      - Directory-level orchestrator
                         - Parses all reports in sequence
                         - Comprehensive summary statistics
                         - JSON export capability

Data Flow:
  Raw Text File (fixed-width) → Parser Module → Normalized Dict → Results Dict

================================================================================
KEY FEATURES
================================================================================

1. Robust Fixed-Width Parsing
   - Efficient string slicing for known positions
   - Fallback regex for flexible pattern matching
   - Handles variable line lengths gracefully

2. Multi-Level Aggregation Support
   - Batch-level (EP-210B)
   - File-level (EP-210C)
   - CPD-level (EP-210D)
   - Run-level (EP-210E)
   - Currency-level (EP-211C/D/E)

3. Large File Optimization
   - Streaming approach for EP-707 (23MB)
   - Memory-efficient record grouping
   - Line-by-line processing where beneficial

4. Comprehensive Record Grouping
   - AD3103 transactions grouped with related records
   - 0500 as transaction header, 0501/0505/0507 as detail records
   - Hierarchical structure preservation

5. Error Resilience
   - Graceful handling of missing fields (returns None)
   - UTF-8 encoding with fallback for special characters
   - Continues parsing on malformed lines
   - Tracks failed files in summary

6. Extensible Design
   - Module-per-report-type for maintainability
   - Consistent data structure output
   - Easy to add new report types
   - Plugin-style sub-report parsing for VSS

================================================================================
USAGE EXAMPLES
================================================================================

Quick Summary:
  from parsers.package_parser import get_package_summary
  summary = get_package_summary("/path/to/settlement/package/")
  print(f"Transactions: {summary['transaction_count']}")

Parse Specific Report:
  from parsers.ep_transactions import parse_ep_705
  result = parse_ep_705("/path/to/EP705.TXT")
  for trans in result['transactions']:
    print(trans['merchant_name'], trans['destination_amount'])

Full Package Parse:
  from parsers.package_parser import parse_package
  results = parse_package("/path/to/settlement/package/")
  transactions = results['transaction_reports']['ep_705']['transactions']

Export to JSON:
  from parsers.package_parser import parse_package, export_to_json
  results = parse_package("/path/to/settlement/package/")
  export_to_json(results, "output.json")
