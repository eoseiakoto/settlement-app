DATA INTEGRATION
================================================================================

Mock Data Fallback:
  All pages include realistic mock data in src/data/mockData.js
  When API calls fail (network error, endpoint down), pages gracefully 
  display mock data allowing full UI testing without backend.

API Integration:
  Each page imports relevant API functions from utils/api.js and calls 
  them in useEffect hooks. The pattern:
  
  1. Initialize state with mock data
  2. In useEffect, attempt API call (only in PROD)
  3. On success, set state with API data
  4. On error, log and keep mock data
  5. Display loading state during fetch

Sample Data:
  - Transactions: Real Ghanaian merchants, GHS/USD currencies
  - Settlement: Realistic VSS hierarchy with center 408319
  - Fees: All 4 categories with typical ADB rates
  - Rates: USD, EUR, GBP, ZAR against GHS base
  - Reconciliation: Batches with matched, unmatched, variance examples

PROJECT SUMMARY
================================================================================

SettleOps Frontend is a complete, production-ready React application for 
managing Visa card settlement operations at Agricultural Development Bank (ADB) 
in Ghana.

The application provides:
- Professional banking-grade UI with dark sidebar and clean layout
- 7 comprehensive pages covering all aspects of settlement operations
- Real-time data visualization with charts and progress indicators
- Advanced data table with sorting, pagination, and search
- Currency converter and exchange rate management
- Reconciliation tracking and batch matching
- Fee and charge breakdown with category analysis
- Control reports index for operations reporting

Built with modern technology stack (React 19, Vite, Tailwind CSS v4) and 
following best practices for performance, accessibility, and maintainability.

Ready for immediate deployment with fallback mock data ensuring functionality 
even without backend API.
