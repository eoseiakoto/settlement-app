API ENDPOINTS IMPLEMENTED:
==========================

Upload & Parse:
  POST   /api/upload                   - Upload ZIP or directory path
  GET    /api/packages                 - List all parsed packages
  GET    /api/packages/<id>            - Get package summary

Transaction Data:
  GET    /api/transactions             - List with filtering/pagination
  GET    /api/transactions/<id>        - Get transaction detail
  GET    /api/transactions/summary     - Aggregated statistics

Settlement Data:
  GET    /api/settlement               - Settlement summary
  GET    /api/settlement/hierarchy     - VSS-100-W hierarchy data
  GET    /api/settlement/interchange   - VSS-120 interchange values
  GET    /api/settlement/fees          - VSS-130, VSS-140, VSS-210 fees
  GET    /api/settlement/reconciliation - VSS-900 series reconciliation

Control & Operations:
  GET    /api/control/batch-summary    - EP-210 batch summaries
  GET    /api/control/reconciliation   - EP-220 reconciliation data
  GET    /api/control/index            - EP-999 report index
  GET    /api/control/currency-rates   - EP-756 currency rates

Dashboard:
  GET    /api/dashboard                - Aggregated metrics for UI

Health & Root:
  GET    /api/health                   - Health check
  GET    /                             - API info & endpoint listing

DATA STORAGE STRUCTURE:
=======================

In-memory dictionary (data_store):
  - packages[]        : Parsed report packages metadata
  - transactions[]    : Transaction records from EP reports
  - settlement{}      : Settlement summary data
  - control{}         : Control/operations data
  - raw_reports{}     : References to raw parsed reports

Ready for SQLite persistence layer addition in production.
